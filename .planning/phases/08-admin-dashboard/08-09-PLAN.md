  <how-to-verify>
**Prerequisites:**
- Backend running: `cd backend && pnpm dev`
- Frontend running: `cd frontend && pnpm dev`
- Admin user exists in database (add to admins table manually if needed)

**Step 1: Admin Access**
1. Login as admin user at http://localhost:3001/login
2. Navigate to http://localhost:3001/admin
3. Verify: Admin layout loads with sidebar
4. Verify: Non-admin users are redirected to /dashboard

**Step 2: Dashboard Metrics**
1. On /admin page, verify 4 metric cards display
2. Check: Total Members shows correct count
3. Check: Total Revenue shows IDR amount
4. Check: Active Subscriptions count
5. Check: Recent Orders (last 30 days) count

**Step 3: Course Management**
1. Navigate to /admin/courses
2. Verify: Course list displays in table
3. Click "Add Course" - form should open
4. Create a test course (title, description, check Free Preview)
5. Verify: Course appears in list
6. Click Edit - form should pre-populate
7. Update title and save
8. Delete the test course

**Step 4: Research Management**
1. Navigate to /admin/research
2. Verify: Research list displays
3. Create research with stock fields (symbol: AAPL, rating: Buy)
4. Verify: Rating badge shows correct color
5. Edit and delete test research

**Step 5: Template Management**
1. Navigate to /admin/templates
2. Verify: Template list displays with file sizes
3. Upload a test file (any small file)
4. Verify: File uploads and appears in list
5. Edit metadata (title, description)
6. Delete test template

**Step 6: User Management**
1. Navigate to /admin/users
2. Verify: User list displays with tier badges
3. Search for a user by email
4. Click to view user details
5. Change user tier from free to member (or vice versa)
6. Verify: Tier badge updates
7. Check payment history displays

**Step 7: Order Management**
1. Navigate to /admin/orders
2. Verify: Order list displays with status badges
3. Filter by status (e.g., "settlement")
4. Verify: Filter applies correctly
5. Check: Amounts display in IDR format
6. Check: Customer info (name, email) visible
7. Test pagination if many orders

**Expected Results:**
- All pages load without errors
- Data tables sort and paginate correctly
- Forms validate and submit successfully
- Status badges have appropriate colors
- Indonesian locale used for dates and currency
